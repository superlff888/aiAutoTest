# 简历整体结构推荐

## 1. 推荐板块顺序（社招 3-5 年）

```
① 个人信息       一行（姓名 / 电话 / 邮箱 / 城市 / 求职岗位）
② 求职意向       一行（目标岗位 + 期望城市 + 期望薪资可选）
③ 核心技能       6-10 项分类列出（用 project-positioning.md 中的模板）
④ 工作经历       倒序，每段含公司 / 时间 / 岗位 / 一句话职责 + 2-3 个量化成果
⑤ 项目经验 ⭐    重头戏，3 个项目用 STAR 模板，占简历 50% 篇幅
⑥ 教育经历       一行（学校 / 学历 / 专业 / 时间）
⑦ 自我评价       可选，写就写 3 行内的硬实力，不要写软实力
```

## 2. 应届 / 1-3 年顺序

```
① 个人信息
② 求职意向
③ 教育经历       (放前面，是应届主要资本)
④ 核心技能
⑤ 项目经验 ⭐⭐  (占 60% 篇幅，是应届最重要的部分)
⑥ 实习经历
⑦ 论文/竞赛/开源 (有就写，加分项)
```

## 3. 5-8 年 / 资深顺序

```
① 个人信息
② 求职意向
③ 核心技能 / 技术雷达
④ 工作经历 ⭐    (重点，每段经历可写多达 5-6 个成就)
⑤ 项目经验      (挑 2-3 个最有代表性的)
⑥ 团队管理经验  (有则单独成段)
⑦ 教育经历      (一行带过)
⑧ 自我评价      (可省)
```

## 4. 一页简历的字数控制（社招 3-5 年模板）

| 板块 | 字数（约） |
|------|-----------|
| 头部 | 50 |
| 核心技能 | 150 |
| 工作经历 | 300 |
| 项目经验 ×3 | 800 |
| 教育 | 30 |
| **合计** | **~1330 字** |

> 一页 A4 中文约 1300-1500 字。**多了说明废话太多，砍。**

## 5. 板块字数动态调整

| 经验年限 | 项目经验 | 工作经历 | 教育 | 总长度 |
|---------|---------|---------|------|--------|
| 应届 | 60% | 10% | 20% | 1 页 |
| 1-3 年 | 50% | 25% | 10% | 1 页 |
| 3-5 年 | 50% | 30% | 5% | 1-2 页 |
| 5-8 年 | 30% | 50% | 5% | 2 页 |
| 8 年+ | 20% | 60% | 5% | 2 页 |

## 6. 简历版面建议

| 项 | 推荐设置 |
|----|---------|
| 字体 | 中文：微软雅黑 / 思源黑体 / 苹方；英文：Helvetica / Arial |
| 字号 | 正文 10-10.5pt / 小标题 11pt / 大标题 13-14pt |
| 行距 | 1.15 - 1.25（不要太密也不要太松） |
| 页边距 | 上下 1.5cm，左右 2cm |
| 加粗 | 只加粗**项目名称 + 数字 + 核心技术名词**，全文加粗 ≤ 20% |
| 颜色 | 主色调 1 种（深蓝/深灰），不超过 2 种 |
| 分割线 | 板块间用细横线分隔，避免装饰图形 |
| 头像 | 国内：放或不放看公司风格；外企：**绝对不放** |
| 个人照片 | 同上 |

## 7. PDF 输出规范

- 格式：**PDF（不要 docx，排版会跑）**
- 文件名：`姓名_目标岗位_城市_X年.pdf`
  - 示例：`张三_AI测试工程师_深圳_5年.pdf`
- PDF 属性：标题填上你的名字（PDF reader → 文档属性 → Title）
- 文件大小：< 2MB（HR 邮箱常有附件大小限制）
- 测试可读：在手机和电脑分别打开，确认排版不跑

## 8. 多版本管理建议

针对**不同岗位**准备多版本简历，每版调整关键词命中率：

| 版本 | 核心调整 |
|------|---------|
| **AI 测试工程师版** | 突出 LLM Eval、promptfoo、RAGAS、Prompt 测试 |
| **LLM 应用工程师版** | 突出 LangChain、Agent、RAG 项目深度 |
| **算法工程师版** | 突出训练、微调、Benchmark、论文 |
| **测试架构师版** | 突出体系建设、团队管理、流程规范 |

> 投递前对照目标 JD 调整，每版投递不同公司。
