# 雷区清单 · 简历自杀行为速查

> **核心原则**（鱼皮版）：**"砍掉虚的，加上实的。'提升用户体验'这种话直接删。"**

## 1. ⚠️ 文案雷区（说了等于没说）

| ❌ 不要写 | ✅ 应该写 |
|----------|----------|
| "参与 XX 项目" | "**主导** / **独立完成** XX 项目" |
| "**负责** 用例编写" | "设计并实现 XX 模块，覆盖 N 个场景" |
| "**协助** 团队优化" | "推动 XX 流程改造，效率提升 X%" |
| "**有一定的** XX 经验" | "**熟练使用** XX 完成 N 个项目" |
| "**了解** Python" | "**3 年 Python 开发经验，主导 N 个 Agent 项目**" |
| "**良好的** 沟通能力" | 删除（HR 不看这个） |
| "**积极主动** / **团队合作**" | 删除（说了等于没说） |
| "**热爱学习**" | 删除（用项目和成果证明） |
| "**精通** Transformer" | "**熟悉** Transformer 架构，参与过 N 次 LoRA 微调实战" |
| "**深入研究** 大模型" | "**完成 X 个 LLM 项目，包括 RAG/Agent/Eval 全链路**" |

## 2. ⚠️ 结构雷区

- ❌ **项目顺序按时间倒序，结果最新最弱的放最前** → ✅ **按"重要性"排序**，把最能体现 AI 能力的放最前
- ❌ **教育经历占半页** → ✅ 一行带过（学校 + 学历 + 专业 + 时间）
- ❌ **技能列表 30+ 项** → ✅ 控制在 10-15 个**核心技能**，按类别分组
- ❌ **简历超过 2 页**（应届 1 页、社招 1-2 页）
- ❌ **混用中英文术语**（"我使用 langchain 框架做了 AI agent 的 testing"）→ ✅ 统一为英文专有名词 + 中文动词
- ❌ **大段文字描述项目** → ✅ Bullet Point + 加粗关键词
- ❌ **每段经历都写 5+ 条 bullet** → ✅ 3-5 条核心成就，宁缺勿滥

## 3. ⚠️ ATS 系统雷区（简历筛选机器人）

- ❌ **PDF 用图片做版面**（OCR 识别不出文字）→ ✅ **文字 PDF**（可复制选中）
- ❌ **简历名 "我的简历.pdf"** → ✅ **"姓名_岗位_城市_年限.pdf"**
- ❌ **用表格做版面**（部分 ATS 解析错乱）→ ✅ 用文本 + 项目符号
- ❌ **使用花哨字体 / 双栏布局** → ✅ 单栏 + 微软雅黑/思源黑体
- ❌ **关键词只写一次** → ✅ 在「技能」「项目」「自我评价」**三处都出现**核心关键词
- ❌ **PDF 元数据为空** → ✅ 设置正确的标题（PDF 属性 → Title）
- ❌ **嵌入特殊符号 / Emoji** → ✅ ATS 可能解析为乱码，少用

## 4. ⚠️ 面试雷区（写了简历就要扛得住追问）

| 写了... | 必须能答上 |
|---------|-----------|
| LangGraph | StateGraph 节点/边/状态管理、Send API 并行机制 |
| RAG | 分块策略、Embedding 模型选择、Top-K、Rerank 原理 |
| Agent | ReAct vs Plan-and-Execute、Tool Use 错误处理 |
| MCP | MCP 协议消息格式、Server/Client 实现差异 |
| LoRA / SFT | 参数量计算、Rank 选择、训练显存估算 |
| 向量数据库 | HNSW vs IVF 索引、召回评估指标 |
| Prompt Engineering | Few-shot 数量影响、CoT 触发条件、温度策略 |
| 评估框架 | LLM-as-Judge 偏差、RAGAS 指标含义、对抗测试设计 |

> **铁律**：**不熟的别写**。简历是双刃剑，写上去就是面试官追问的入口。

## 5. ⚠️ 数字雷区（数学不可能 / 不可信）

- ❌ "**用户量提升 1000%**" → 仅在从 1 → 11 这种小基数下合理，写实数 + 倍数
- ❌ "**召回率提升 200%**" → 召回率 [0,100%]，绝对不可能 +200pp
- ❌ "**完美**解决 / **零** Bug" → 没有零 Bug 的系统
- ❌ "**所有** 项目都做过" → "所有"是危险词
- ❌ **个人简历写月调用 100 亿次** → 个人项目不可能达到这量级，可信度归零
- ❌ **百分比和绝对值脱节** → "提升 99%"（从 1% 到 1.99%）虽然数学对，但容易被质疑

## 6. ⚠️ 自我评价雷区

写自我评价**只允许**这两种情况：
- ✅ 你有岗位匹配度高的硬实力，需要总结概括（"5 年测试经验 + 2 年 LLM 实战，主导过 X 个项目，期望负责 AI 测试体系建设"）
- ✅ 你有特殊背景需要解释（"从开发转测试，对代码白盒测试有独特视角"）

**禁止写**：
- ❌ "我是一个 XX 的人"（标签化空话）
- ❌ "对技术充满热情"（不写也默认你有）
- ❌ "希望加入贵公司一起成长"（求职信而非简历内容）
- ❌ 罗列性格特征（细心、踏实、勤奋）

- [ ] 没有"团队合作 / 积极主动"这类废话

- [ ] 总长度 ≤ 2 页
- [ ] 写的每个技术词都能扛 3 轮追问
- [ ] 项目里至少有 1 个 **LangGraph / Agent / RAG** 关键项目
- [ ] 求职意向、技能、项目三处都出现核心关键词
