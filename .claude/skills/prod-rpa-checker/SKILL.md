# prod-rpa-checker 技能说明

## 功能概述

校验各交易中心数据是否采集到最新且日期无中断：

1. **最新数据时间验证**：根据 JSON 配置，验证数据是否采集到最新（如 D-1、D-2）
2. **日期连续性验证**：验证指定时间范围内每一天的数据都存在

## 数据中心配置

- **JSON配置**：[数据中心类型定义.json](doc/数据中心类型定义.json)
- **MD文档**：[数据中心类型定义.md](doc/数据中心类型定义.md)

配置结构：

```json
{
  "广东": {
    "trade_center_id": 1,
    "日前节点电价": "D",
    "实时节点电价": "D-2",
    ...
  },
  ...
}
```

## SQL模板

SQL文件位于 `sqls/` 目录，使用占位符：

- `{{trade_center_id}}` - 交易中心ID
- `{{offset_days}}` - D加减天数（D-1=1, D-2=2, D+1=-1, D=0）

| SQL文件              | 对应数据类型     |
| -------------------- | ---------------- |
| 代理用户用电明细.sql | 代理用户用电明细 |
| 代理用户用电总计.sql | 代理用户用电总计 |
| 日前成交电量.sql     | 日前成交电量     |
| 实时节点电价.sql     | 实时节点电价     |
| 实时结算电价.sql     | 实时结算电价     |
| 网侧实际.sql         | 网侧实际         |
| 网侧预测.sql         | 网侧预测         |
| 日前节点电价.sql     | 日前节点电价     |
| 日前结算电价.sql     | 日前结算电价     |

## 参数转换规则

D加减时间转换为 offset_days：

- D → 0
- D-1 → 1
- D-2 → 2
- D-3 → 3
- D+1 → -1
- D+2 → -2

多个时间值（如 D-1/D-2）：优先查询 D-1，查询不到再查 D-2

## 校验流程

### 1. 最新数据时间验证

对于每个交易中心的数据类型：

1. 读取 JSON 配置中的最新数据时间
2. 如果为 "暂未接入"，跳过
3. 如果有多个时间值（如 D-1/D-2），先查 D-1
4. 执行 SQL，查询最大日期
5. 与预期最新数据时间对比：
   - 预期 D，查询到的最大日期应为今天
   - 预期 D-1，查询到的最大日期应为昨天
   - 如不符，断言失败并说明原因

### 2. 日期连续性验证

对于指定时间范围（如月初至今）：

1. 生成日期序列
2. 查询数据库中的所有日期
3. 对比找出缺失的日期
4. 如有缺失，断言失败并列出缺失日期

## 执行示例

用户输入：

```
检查广东的日前节点电价数据
```

技能执行：

1. 读取 JSON 配置，广东的日前节点电价最新数据时间为 D
2. 获取 trade_center_id = 1
3. offset_days = 0
4. 读取 [日前节点电价.sql](sqls/日前节点电价.sql)
5. 替换参数执行 SQL
6. 验证最大日期是否为今天
7. 返回验证结果

## 输出格式

验证通过：

```
✅ 广东 - 日前节点电价：数据最新（2026-05-14）
✅ 广东 - 实时节点电价：数据最新（2026-05-12，D-2）
```

验证失败：

```
❌ 广东 - 日前节点电价：数据未采集到最新
   预期：2026-05-14（D）
   实际：2026-05-13（D-1）

❌ 广东 - 日前成交电量：日期不连续
   时间范围：2026-05-01 至 2026-05-14
   缺失日期：2026-05-02, 2026-05-05, 2026-05-08
```

## 注意事项

- 暂未接入的数据类型会被跳过，不进行校验
- 多时间值的数据类型，优先查询较新时间，查询不到再查较旧时间
- 断言失败时需明确指出预期与实际的差异
- 日期缺失时需列出具体的缺失日期
