# P1-3 提示词内容修复（状态迁移法优先级）

> **状态**：✅ 已确定
> **优先级**：P1（强烈建议）
> **影响文件**：`pormpts/generate_case_prompt.py`

---

## 1. 问题定位

`generate_case_prompt.py` 内部两处自相矛盾：

| 位置 | 行号 | 内容 | 优先级 |
|------|------|------|--------|
| 方法 e（状态迁移法） | 62-63 | "对需求文档未允许的转移路径...生成用例验证系统拒绝该操作" | P1 |
| 约束规则 4 | 88 | "P2 = 非法状态迁移..." | P2 |

**影响**：模型看到两处定义，会以后一次出现的为准（约束规则在方法 e 之后），实际生成会偏向 P2。但示例 7、8 都把非法状态迁移归为 P1——示例与约束不一致。

---

## 2. 修复策略

**统一为 P1**（与示例 7、8 一致），并把约束规则 4 改写得更清晰。

---

## 3. 代码改动

[generate_case_prompt.py](../rag_agent/pormpts/generate_case_prompt.py) 第 88 行修改：

**修改前**：

```python
4. 优先级标准：P0 = 核心主流程/基本流场景/有效等价类正向用例；P1 = 备选流场景/边界值/无效等价类/合法状态迁移/判定表组合；P2 = 边缘异常场景/容错性测试（仅在需求明确提到容错/回滚/降级时生成，无依据则不生成）。
```

**修改后**：

```python
4. 优先级标准：
   - P0 = 核心主流程/基本流场景/有效等价类正向用例
   - P1 = 备选流场景/边界值/无效等价类/合法状态迁移/非法状态迁移/判定表组合
   - P2 = 边缘异常场景/容错性测试（**仅在需求明确提到容错/回滚/降级时生成，无依据则不生成**）
```

## 4. 验证方法

跑同一份状态迁移需求（订单状态），统计"非法迁移"用例的 priority 分布：

```python
# quick_test_p1_3.py
from langchain_core.messages import HumanMessage
from models import llm_model_generate
from rag_agent.pormpts.generate_case_prompt import get_generate_case_prompt

req = """
订单状态流转：待支付→已支付→已发货→已签收。
允许用户在待支付时取消订单。
"""

messages = get_generate_case_prompt(req)
response = llm_model_generate.invoke(messages)
content = response.content

# 简单解析
import re
priorities = re.findall(r'"priority":\s*"(P[012])"', content)
print("优先级分布:", {p: priorities.count(p) for p in set(priorities)})

# 修复后：非法状态迁移用例应全部 P1
# 修复前：可能被错误归为 P2
```

修复前后的对照观察：
- 修复前：可能看到 P2 比例 > 0（违反约束）
- 修复后：非法迁移用例应**全部 P1**
