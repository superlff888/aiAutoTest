## 2. 核心方案：结构化输出兼容设计

> 用户决策（2026-06-14）：采用**精简后的混合调用方案**
> - 必须**带思考**（模型层不能关思考）
> - 保留 `format_result` 函数
> - 保留 `with_structured_output` 的 Pydantic 强校验能力
> - 用 `try/except` 兜底
> - 用 type check 健壮性检查

**核心入口**：[`hybrid_structured_invoke(llm, messages, schema)`](06-结构化输出兼容方案.md)

**两条路径**：
```
路径 A (思考型模型 - 带 <think> 标签):
  llm.invoke(messages)  →  format_result(content)  →  schema.model_validate(dict)

路径 B (非思考型模型 - 无 <think> 标签):
  llm.with_structured_output(schema).invoke(messages)
  (LangChain 内部走 tool calling, 自动按 schema 解析)
```

**完整防御链**：路径 A → 路径 B → 二次兜底（手动 invoke + format_result + model_validate）
