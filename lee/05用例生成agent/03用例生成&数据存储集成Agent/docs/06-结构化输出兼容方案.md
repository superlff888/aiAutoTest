# 核心方案：结构化输出兼容设计（hybrid_structured_invoke）

> **状态**：✅ **已确定**（用户决策：2026-06-14）
> **优先级**：核心方案
> **影响文件**：`common/hybrid_structured_invoke.py`（**新建**）、`common/model_output_to_json.py`、`workflow/case_generator.py`

---

## 1. 决策背景

| 约束 | 说明 |
|------|------|
| ✅ **必须带思考** | 模型层不能关掉 `<think>` 标签（用户已确认） |
| ✅ **保留 `format_result`** | 不放弃字符串解析能力 |
| ✅ **保留 `with_structured_output`** | 用 Pydantic 强校验 |
| ✅ **`try/except` 兜底** | 健壮性 |
| ✅ **健壮性检查** | type check 等 |

**三者兼容思路**：
```
1. 先 llm.invoke 拿 raw content (含可能的 <think>)
2. 健壮性检查: 是否带 <think> 标签
3a. 带思考 (路径A) → format_result 解析为 dict → model_validate 强校验
3b. 无思考 (路径B) → with_structured_output 强制 schema
4. 任一失败, 兜底到对方路径
```

---

## 2. 架构图

```
hybrid_structured_invoke(llm, messages, CaseList)
  │
  ├─ Step 1: llm.invoke(messages)
  │    └─ 异常? → ValueError("LLM 调用失败")
  │
  ├─ Step 2: 检测 <think> 标签
  │    │
  │    ├─ 命中 (思考型) → 路径 A
  │    │    ├─ format_result(content)
  │    │    │    └─ 失败 / 类型错 → 降级路径 B
  │    │    ├─ schema.model_validate(dict) ← Pydantic 强校验
  │    │    │    └─ ValidationError → 降级路径 B
  │    │    └─ ✅ 返回 BaseModel
  │    │
  │    └─ 未命中 (非思考型) → 路径 B
  │
  └─ 路径 B: _invoke_via_structured_output()
       ├─ llm.with_structured_output(schema).invoke(messages)
       │    └─ 失败 → 二次兜底
       ├─ 二次兜底: llm.invoke + format_result + model_validate
       │    └─ 失败 → ValueError("全部路径失败")
       └─ ✅ 返回 BaseModel
```

### 3.1 `common/hybrid_structured_invoke.py`（**新建**）

```python
# !/usr/bin/env python3,# -*- coding: utf-8 -*-
# --------------------------------------------
# @FilePath    : lee\05用例生成agent\03用例生成+数据存储集成Agent\rag_agent\common\hybrid_structured_invoke.py
# @Author      : Lee大侠
# @Desc        : 兼容思考型模型的混合结构化调用入口
#                路径A: 思考型 → format_result + Pydantic model_validate
#                路径B: 非思考型 → LangChain with_structured_output
#                兜底: 任一失败自动降级
# @CreateTime  : 2026/06/14
# ========================================================


import logging
from typing import Type, TypeVar

from pydantic import BaseModel, ValidationError

from rag_agent.common.model_output_to_json import format_result

logger = logging.getLogger(__name__)

T = TypeVar("T", bound=BaseModel)


def _has_thinking_tags(content: str) -> bool:
    """健壮性检查 1: 是否带 <think>...</think> 标签"""
    if not content or not isinstance(content, str):
        return False
    return "<think>" in content and "</think>" in content


def _is_valid_parsed(parsed) -> bool:
    """健壮性检查 2: 解析结果是否合法（dict 或 list）"""
    return isinstance(parsed, (dict, list))


def hybrid_structured_invoke(
    llm,
    messages: list,
    schema: Type[T],
) -> T:
    """
    兼容思考型模型的混合结构化调用

    路径 A (思考型模型 - 带 <think> 标签):
        llm.invoke(messages) → format_result(content) → schema.model_validate(dict)

    路径 B (非思考型模型 - 无 <think> 标签):
        llm.with_structured_output(schema).invoke(messages)
        (LangChain 内部走 tool calling, 自动按 schema 解析)

    两种路径都得到 Pydantic 强校验。
    路径 A 失败时降级到路径 B;路径 B 失败时降级到二次兜底。

    :param llm: LangChain chat model
    :param messages: list of BaseMessage
    :param schema: 目标 Pydantic 模型类
    :return: schema 的实例
    :raises: ValueError 当全部路径都失败时
    """
    # ===== 1. 先 invoke 拿 raw content (统一入口) =====
    try:
        response = llm.invoke(messages)
        raw_content = response.content or ""
    except Exception as e:
        raise ValueError(f"LLM 调用失败: {type(e).__name__}: {e}")

    # ===== 2. 健壮性检查: 是否带思考标签 =====
    has_thinking = _has_thinking_tags(raw_content)
    logger.debug(f"[hybrid] has_thinking={has_thinking}, raw_len={len(raw_content)}")

    # ===== 3a. 路径 A: 思考型模型 =====
    if has_thinking:
        try:
            # 步骤 1: format_result 剥 <think> + 解析为 dict
            parsed = format_result(raw_content)
            # 健壮性检查 2: parsed 必须是 dict 或 list
            if not _is_valid_parsed(parsed):
                raise ValueError(
                    f"format_result 解析结果类型不合法: {type(parsed).__name__}"
                )
            # 步骤 2: Pydantic 强校验
            return schema.model_validate(parsed)
        except (ValidationError, ValueError) as e:
            logger.warning(f"[hybrid] 路径 A 失败, 降级到路径 B: {e}")
            return _invoke_via_structured_output(llm, messages, schema)

    # ===== 3b. 路径 B: 非思考型模型 =====
    else:
        return _invoke_via_structured_output(llm, messages, schema)


def _invoke_via_structured_output(llm, messages: list, schema: Type[T]) -> T:
    """
    通过 LangChain 原生 with_structured_output 调用
    这是路径 B, 也是路径 A 失败后的兜底
    """
    try:
        structured_llm = llm.with_structured_output(schema)
        return structured_llm.invoke(messages)
    except Exception as e:
        logger.warning(
            f"[hybrid] with_structured_output 失败, 兜底到 format_result+model_validate: {e}"
        )
        # 二次兜底: 手动 invoke + format_result + model_validate
        try:
            response = llm.invoke(messages)
            parsed = format_result(response.content or "")
            if not _is_valid_parsed(parsed):
                raise ValueError(f"兜底解析结果类型不合法: {type(parsed).__name__}")
            return schema.model_validate(parsed)
        except Exception as e2:
            raise ValueError(
                f"hybrid_structured_invoke 全部路径失败。"
                f"最后错误: {type(e2).__name__}: {e2}"
            )
```

### 3.2 `common/model_output_to_json.py`（加类型 hint，保持兼容）

```python
# !/usr/bin/env python3,# -*- coding: utf-8 -*-
# --------------------------------------------
# @FilePath    : lee\05用例生成agent\03用例生成+数据存储集成Agent\rag_agent\common\model_output_to_json.py
# @Author      : Lee大侠
# @Desc        : LLM 输出解析工具(供 hybrid_structured_invoke 调用)
# @UpdateTime  : 2026/06/14
# ========================================================


import json
import re


def format_result(result: str) -> dict | list:
    """
    解析 LLM 输出为 Python 对象
    兼容: <think>...</think> 标签、```json``` 标记、纯 JSON 字符串
    """
    if not result or not isinstance(result, str):
        raise ValueError(f"format_result 输入必须是非空字符串, 当前类型: {type(result)}")

    # 1) 剥思考标签
    if "<think>" in result and "</think>" in result:
        result = result.split("</think>", 1)[1]

    # 2) 剥 markdown 标记
    result = result.replace('```json', '').replace('```', '').strip()

    # 3) JSON 解析
    parsed = json.loads(result)  # 可能是 dict 或 list

    # 4) 类型检查
    if not isinstance(parsed, (dict, list)):
        raise ValueError(
            f"format_result 解析结果非 dict/list, 当前: {type(parsed).__name__}"
        )
    return parsed
```

### 3.3 `workflow/case_generator.py` 三个节点改造

```python
# case_generator.py 顶部新增 import
from rag_agent.common.hybrid_structured_invoke import hybrid_structured_invoke
# 删除原来的 from rag_agent.common.model_output_to_json import format_result


class GenerateCaseWorkflow:

    @staticmethod
    def _generate_case(state: State):
        """用例生成节点 - 路径A/B 混合"""
        requirements = state.get('requirements')
        messages = generate_case_prompt.get_generate_case_prompt(requirements)
        try:
            result: CaseList = hybrid_structured_invoke(
                llm_model_generate, messages, CaseList
            )
            cases = [c.model_dump() for c in result.cases]
            print(f"用例生成完毕,当前一共生成了{len(cases)}条用例")
            return {"generated_cases": cases}
        except Exception as e:
            print(f"生成用例失败: {e}")
            return {"generated_cases": []}

    @staticmethod
    def _review_case(state: State):
        """用例评审节点 - 路径A/B 混合"""
        _case = state.get('current_review_case')
        requirements = state.get('requirements')
        messages = case_review_prompt.get_review_prompt(requirements, _case)
        try:
            result: ReviewResult = hybrid_structured_invoke(
                llm_model_review, messages, ReviewResult
            )
            result_json = result.model_dump()
        except Exception as e:
            print(f"评审失败,按不通过处理: {e}")
            result_json = {
                "review_result": "不通过",
                "review_desc": "LLM 输出解析失败",
                "failed_dimensions": ["评审失败"],
            }

        if result_json.get('review_result') == "通过":
            _case["review_result"] = "通过"
            _case["review_desc"] = result_json.get('review_desc', '')
            _case["failed_dimensions"] = result_json.get('failed_dimensions', [])
            return {"review_passed_cases": [_case]}
        _case["review_result"] = "未通过"
        _case["review_desc"] = result_json.get('review_desc', '')
        _case["failed_dimensions"] = result_json.get('failed_dimensions', [])
        return {"review_failed_cases": [_case]}

    @staticmethod
    def verify_coverage(state: State):
        """覆盖率检查节点 - 路径A/B 混合"""
        review_passed_cases = state.get('review_passed_cases', [])
        requirements = state.get('requirements')
        messages = verify_coverage_prompt.get_verify_coverage_prompt(
            requirements, review_passed_cases
        )
        try:
            result: CoverageResult = hybrid_structured_invoke(
                llm_model_coverage, messages, CoverageResult
            )
            result_dict = result.model_dump()
        except Exception as e:
            print(f"覆盖率检查失败: {e}")
            result_dict = {
                "coverage_report": "LLM 解析失败",
                "coverage": "0.0%",
                "recommend": [],
            }
        print("覆盖率:", result_dict.get('coverage'))
        return {"coverage_check_result": result_dict}
```

## 4. 健壮性检查清单

| # | 检查项 | 实现位置 | 失败处理 |
|---|--------|---------|---------|
| ① | LLM 调用本身成功 | `hybrid_structured_invoke` 第 1 步 | 直接抛 `ValueError` |
| ② | 响应 content 非空 | `raw_content = response.content or ""` | 走路径 B（`with_structured_output` 会报错） |
| ③ | 思考标签检测 | `_has_thinking_tags()` | 决定走路径 A 或 B |
| ④ | `format_result` 解析成功 | `format_result()` 内 `json.loads` | 路径 A 失败 → 降级到路径 B |
| ⑤ | 解析结果类型合法 | `_is_valid_parsed()` | 路径 A 失败 → 降级到路径 B |
| ⑥ | Pydantic 校验通过 | `schema.model_validate()` | 路径 A 失败 → 降级到路径 B |
| ⑦ | `with_structured_output` 成功 | `_invoke_via_structured_output` 第 1 步 | 二次兜底到 `format_result` |
| ⑧ | 二次兜底成功 | 二次 invoke + 解析 + validate | 抛 `ValueError` 含完整错误信息 |

**8 个检查点形成完整防御链**。
