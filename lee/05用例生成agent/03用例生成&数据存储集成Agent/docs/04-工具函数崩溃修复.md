# P0-4 工具函数崩溃修复

> **状态**：✅ 已确定
> **优先级**：P0（必修）
> **影响文件**：`tools/case_generator_tools.py`

---

## 1. 问题定位

`tools/case_generator_tools.py:55-70`：

```python
state = workflow.invoke({"requirements": requirements, "review_passed_cases": exist_case})
cases = state.get("review_passed_cases")          # cases 是 List[dict]
coverage_report = cases.get("coverage_check_result")  # ❌ List 没有 .get()，会 AttributeError
new_case_count = len(cases) - len(exist_case)     # 这里是 list，OK
for case_ in cases:                               # 迭代 OK
    if case_.get("case_id") in exist_case_ids:    # case_ 是 dict，OK
        continue
    else:
        new_cases.append(case_)
```

**问题清单**：
1. `coverage_report = cases.get(...)` —— 列表没有 `.get()`，必然 `AttributeError`
2. `exist_case` 类型不明确（可能是 `List[GenerateCase]` 也可能是 `List[dict]`）
3. `exist_case_ids = [case.case_id for case in exist_case]` 只支持 Pydantic 实例，不支持 dict

---

## 2. 修复策略

从 `state` 正确读取覆盖率报告，修复增量计数逻辑，统一处理 dict / Pydantic 两种入参。

---

## 3. 代码改动

**全文替换** [tools/case_generator_tools.py](../rag_agent/tools/case_generator_tools.py)：

```python
# !/usr/bin/env python3,# -*- coding: utf-8 -*-
# --------------------------------------------
# @FilePath    : lee\05用例生成agent\03用例生成+数据存储集成Agent\rag_agent\tools\case_generator_tools.py
# @Author      : Lee大侠
# @Desc        : 用例生成 / 补充生成工具
# @UpdateTime  : 2026/06/14
# ========================================================


from typing import List, Union, Dict, Any

from rag_agent.tools.data_model import GenerateCase
from rag_agent.workflow.case_generator import GenerateCaseWorkflow
from langchain.tools import tool


def _get_case_id(case: Union[GenerateCase, Dict[str, Any]]) -> str:
    """兼容 Pydantic 实例和 dict 两种入参"""
    if isinstance(case, GenerateCase):
        return case.case_id
    if isinstance(case, dict):
        return case.get("case_id", "")
    return ""


def _format_generate_result(cases: list, coverage_report: dict) -> str:
    """格式化用例生成结果"""
    return (
        f"用例生成完成。\n"
        f"- 通过评审用例数：{len(cases)}\n"
        f"- 覆盖率：{coverage_report.get('coverage', 'N/A')}\n"
        f"- 覆盖率报告：{coverage_report.get('coverage_report', 'N/A')}\n"
        f"- 仍需补充：{coverage_report.get('recommend', [])}\n"
        f"- 完整用例：{cases}"
    )


def _format_supplement_result(new_cases: list, coverage_report: dict, old_count: int) -> str:
    """格式化补充生成结果"""
    return (
        f"本次补充生成用例完成。\n"
        f"- 补充前用例数：{old_count}\n"
        f"- 补充后用例数：{old_count + len(new_cases)}\n"
        f"- 本次新增用例数：{len(new_cases)}\n"
        f"- 新增用例数据：{new_cases}\n"
        f"- 最新覆盖率：{coverage_report.get('coverage', 'N/A')}\n"
        f"- 覆盖率报告：{coverage_report.get('coverage_report', 'N/A')}\n"
        f"- 仍需补充：{coverage_report.get('recommend', [])}"
    )


@tool("测试用例生成", description="根据需求生成测试用例（含评审、覆盖率自检、补充迭代全流程）")
def generate_case(requirements: str) -> str:
    """
    根据需求生成测试用例
    :param requirements: 功能的需求文档
    :return: 生成结果摘要
    """
    workflow = GenerateCaseWorkflow().create_generate_case_workflow()
    state = workflow.invoke({"requirements": requirements})

    cases = state.get("review_passed_cases", []) or []
    coverage_report = state.get("coverage_check_result", {}) or {}

    return _format_generate_result(cases, coverage_report)


@tool("补充生成用例", description="根据已有的测试用例，补充生成测试用例")
def generate_case_by_exist(requirements: str, exist_case: List[Dict[str, Any]]) -> str:
    """
    根据已有的测试用例，补充生成测试用例
    :param requirements: 功能的需求文档
    :param exist_case: 已有的测试用例列表（dict 格式或 Pydantic 实例）
    :return: 补充生成结果摘要
    """
    workflow = GenerateCaseWorkflow().create_supplement_generate_workflow()

    # 记录补充前的 case_id 集合（兼容 Pydantic 和 dict）
    old_case_ids = {_get_case_id(c) for c in (exist_case or []) if _get_case_id(c)}

    state = workflow.invoke({
        "requirements": requirements,
        "review_passed_cases": [c.model_dump() if isinstance(c, GenerateCase) else c
                                for c in (exist_case or [])],
    })

    # state 里的所有用例（含旧的+新补充的）
    all_cases = state.get("review_passed_cases", []) or []
    coverage_report = state.get("coverage_check_result", {}) or {}

    # 过滤出本次新增的用例
    new_cases = [c for c in all_cases if c.get("case_id") not in old_case_ids]

    return _format_supplement_result(
        new_cases=new_cases,
        coverage_report=coverage_report,
        old_count=len(old_case_ids),
    )


if __name__ == '__main__':
    requirements = """
             F1.1 用户注册
    🧩 功能背景
    新用户通过注册方式创建账户，支持邮箱/用户名+密码的注册方式。
    🚶 主流程
    1. 用户打开注册页，填写注册信息
    2. 系统校验格式与唯一性（用户名、邮箱）
    3. 提交注册，后台创建账户，初始状态为"正常"
    4. 注册成功后自动登录并跳转首页
    ⚠️ 异常流程
    ● 邮箱/用户名已被注册：提示"已存在"
    ● 两次密码不一致：提示用户重新输入
    📌 状态规则
    ● 新用户状态为"正常"
    ● 注册时间记录为创建时间，头像为默认图
    📌 业务规则
    ● 用户名唯一，支持 4~20 位字母数字组合
    ● 密码长度不少于 6 位
    ● 邮箱必须符合格式 xxx@xxx.xx
    """
    result = generate_case_by_exist.invoke({
        "requirements": requirements,
        "exist_case": [],
    })
    print(result)
```

## 4. 修复对照表

| 错误点 | 原代码 | 新代码 |
|--------|-------|-------|
| `coverage_report = cases.get(...)` | ❌ AttributeError | `coverage_report = state.get(...)` |
| `exist_case_ids = [case.case_id ...]` | ❌ 只支持 Pydantic | `_get_case_id()` 兼容两种 |
| `new_case_count = len(cases) - len(exist_case)` | ❌ 实际未使用 | 删除 |
| 格式化输出 | ❌ 散乱 | `_format_*_result()` 抽取 |
